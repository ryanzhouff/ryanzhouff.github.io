DOM元素和DOM结点

Ryan Zhou
2023-12-26
DOM,node,element

DOM

DOM是一个接口(API)，它将 HTML 或 XML 文档视为树形结构，其中每个结点都是文档的对象。DOM 还提供了一组方法来查询树、改变结构、样式。

结点可以附加事件监听器，一旦事件被触发，事件处理函数就会执行。

Node

节点是 DOM 层次结构中任何类型对象的通用名称(元素的更通用名称)。它可以是内置的 DOM 元素，如document或document.body，它还是可以是 HTML 标签，如<input>或<p>，它还可以是文本或注释。

node 有很多种类型，文本结点(text nodes)，注释结点(comment nodes)，文档结点(document nodes)等等：

  • Element Node (Node.ELEMENT_NODE <=> 1)
  • Attribute Node (Node.ATTRIBUTE_NODE <=> 2)
  • Text Node (Node.TEXT_NODE <=> 3)
  • CData Section Node (Node.CDATA_SECTION_NODE <=> 4)
  • Processing Instruction Node (Node.PROCESSING_INSTRUCTION_NODE <=> 7)
  • Comment Node (Node.COMMENT_NODE <=> 8)
  • Document Node (Node.DOCUMENT_NODE <=> 9)
  • Document Type Node (Node.DOCUMENT_TYPE_NODE <=> 10)
  • Document Fragment Node (Node.DOCUMENT_FRAGMENT_NODE <=> 11)

Element

element只是其中一种特定类型的结点，它的nodeType值是1，它可以在 HTML 文档中直接用 HTML 标签指定，如<p>, <body>。

HTMLCollection vs NodeList

HTMLCollection 是 HTML 元素集合，不包含其他类型的 node，只有 element。返回 HTMLCollection 的属性和方法有：

  • Element.children (只包含元素类型的 node)
  • document.getElementsByClassName()

HTMLCollection 是实时的，当 DOM 元素发生变更是它会自动更新。基于此，可以创建一个副本(使用Array.from)来迭代，添加、移动或移除结点。

NodeList 包含所有类型的 node，包括 element。返回 NodeList 的属性或方法有：

  • Node.childNodes (包含所有子结点，包括非元素结点，如文本和注释)
  • document.querySelectorAll()

尽管 NodeList 不是数组，但它可以使用forEach遍历。也可以通过Array.from()方法转换成真正的数组。

NodeList 分为实时 NodeList(live NodeList)和静态 NodeList(static NodeList)。
实时 NodeList 意味着 DOM 的变更会自动更新 NodeList，Node.childNodes返回的 list 就是实时的。与之相反的就是静态 NodeList，DOM 的任何变更都不会影响到集合的内容，document.querySelectorAll()方法返回的就是静态 NodeList

NodeList 实例方法：

  • NodeList.entries(): 返回一个迭代器
  • NodeList.forEach()：用于遍历 NodeList
  • NodeList.keys(): 返回一个迭代器，所有 node 的索引
  • NodeList.values(): 返回一个迭代器

EventTarget

EventTarget 是触发事件的对象。event.target

Element 还有它的子元素，如 Document 和 Window 都是常见的事件目标，但是其他对象也可以是事件目标，如 XMLHttpRequest, AudioNode 和 AudioContext。

实例方法：

  • EventTarget.addEventListener()
  • EventTarget.removeEventListener()
  • EventTarget.dispatchEvent()

addEventListener(type, listener, options?)，其中options对象包括以下属性：

  • capture: 是否在捕获阶段触发事件，默认false
  • once：表明事件监听器(listener)是否只执行一次，如果为true，事件监听器(事件处理函数,listener)会被自动移除，如果未指定，默认为false
  • passive: 如果为 true， preventDefault()会失效。
  • signal: AbortSignal 对象，当指定的 AbortSignal 对象的abort()方法被调用，listener 会被移除。

DocumentFragment

DocumentFragment是轻量级的Document，存储了 DOM 片段，但是它不是 DOM 树的一部分，对DocumentFragment的修改不会影响 DOM 树。

常用场景：创建文档片段，向文档片段插入元素，然后把文档片段插入到 DOM 树中。在 DOM 树中，文档片段会被它的所有子结点替换。

因为文档片段是在内存中的，而且不是主 DOM 树的一部分，所以向它插入结点不会影响页面的 reflow。

创建片段的方式：

文档片段继承自Node结点，所以它可以使用 node 结点的appendChild方法。使用appendChild方法时需要注意：

如果被插入的结点已经位于文档树中，那么appendChild只会将它从原先的位置移动到新位置，也就说，一个结点不能同时出现在文档的不同位置。如果某个结点已经有父节点，在被传递给该方法后，它首先会被移除，再插入到新的位置。如果想要保留已经在文档中的结点，可以先使用Node.cloneNode()方法创建一个副本，再将副本附加到目标父节点下。**cloneNode制作的副本不会自动保持同步**

属性

parentNode 只读，获取结点的父节点，没有返回 null，Document 和 DocumentFragment 没有父节点，始终返回 null

方法

parentNode.insertBefore(newNode, refNode) 将新节点插入到 refNode 的前面，如果新结点在文档中已经存在，insertBefore 方法会将新节点从当前位置移动到新的位置，也就是在插入到新位置之前会将它从原来的位置移除。
