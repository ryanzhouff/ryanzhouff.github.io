函数式编程指南

Ryan Zhou
2023-12-22
fp, functional programming

函数式编程，Functional Programming，简称 FP，是一种编程范式。

它将电脑运算视为函数运算，并且避免使用程序状态以及易变对象。其中，λ 演算（lambda calculus）为该语言最重要的基础。而且，λ 演算的函数可以接受函数当作输入（引数）和输出（传出值）。

比起指令式编程，函数式编程更加强调程序执行的结果而非执行的过程，倡导利用若干简单的执行单元让计算结果不断渐进，逐层推导复杂的运算，而不是设计一个复杂的执行过程。

它涉及很多数学上的学科：

  • 抽象代数
  • 群论
  • 范畴论

但 FP 和 OOP 的关系也不是高低贵贱之分，而是风格差异。
这些思想不是教条，而是前人为了写出更好的代码进行的思考和总结。
在学习 FP 的过程中也一定会碰到和 OOP 在设计模式上的对比。

在实际业务中，经常会和副作用打交道（网络请求，用户事件，I/O 等）

Pure functions

函数式编程语言中的纯粹(purity)指的就是纯函数。

  1. 纯函数对相同的 input 总能产生相同的 output
  1. 纯函数没有 side effect

不纯的函数：

调用它们会修改文件，向服务器发送数据或者调用系统获取 socket，永远无法预测这些函数会返回什么。

因为 pure function 的不可变性，它的结果是可预测，更容易 unit test

函数式编程语言并不能彻底去除 side effect，只能限制它们。
如何处理 side effects? 使用 monad 把 impure 的部分放在程式边界，尽量让大部分 function 是 pure function

高阶函数

在函数式编程中，函数是语言的第一公民。换句话说，函数就是另一种值。

因为函数是值，我们可以把它们当作值传入参数。

高阶函数既可以接受函数作为参数传入，也可以把函数作为返回值返回，或者同时满足两个条件。

限制函数的执行次数

函数组合

在数学中，f.g 是函数的组合，称为函数 f 与函数 g 的复合函数，或者说在 g 之后调用 f。所以(f.g)(x)相当于x为自变量调用函数 g，再以结果为自变量调用函数 f，简写成f(g(x))

管道机制（pipeline）的例子，即前一个函数的输出是后一个函数的输入
