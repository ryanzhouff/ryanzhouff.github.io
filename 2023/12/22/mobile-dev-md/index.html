    CSS 中，rem表示 root 元素的字体大小。
    即，1rem等于 HTML 元素的字体大小，对于大多数的浏览器，默认值是16px。

    viewport 的width属性: viewport 尺寸，设置为device-width时，视口的宽度就是设备的宽度，即100vw或视口宽度的 100%

    window.innerWidth可以获取设备的宽度。
    document.documentElement.clientWidth可以获取视口的宽度。
    当 viewport 的width的值是device-width，设备的宽度等于视口的宽度。

    不同的设备的视口是不同的，所以在不同的设备中html元素的font-size值也不同，所以元素的缩放效果也不一样。

    假设以 iPhone6 为基准，屏幕宽度为 375，那么html元素的font-size可以通过以下方式计算

    通常移动端 UI 设计稿按照 iPhone6 的物理像素为基准进行设计，即 750px，在量取尺寸的时候，需要除以 2 才能适配页面中的 css 逻辑像素

    37.5px就是基准值，和 JS 代码中的保持一致。假如 A 元素在设计稿上的宽度是120px，那么在 iPhone6 上就是1.6rem。

    当手机切换到 iPhone XR 时，html的font-size的值就变成了41.4px，因为 A 元素的宽度是1.6rem，那么它的实际宽度就是41.4 x 1.6 = 66.24，这就起到了缩放效果。

    总结两点：

    • 动态计算html的font-size值
    • 在 CSS 中将 px 转为 rem

    vw

    使用 vw 布局就不需要去设置根元素的 font size 了

    页面所有元素都用 vw 做单位。设计稿750px=100vw，

    在移动端的 Safari 或 Chrome 浏览器中 设置页面 100vh 的，会出现滚动条，原因是移动浏览器中，100vh 是去掉地址栏的高度，所以会出现滚动条

    解决办法是用 100%替代 100vh，或者用 JavaScript 计算

    100%和 100vw 的区别？
    百分比是根据父元素的宽度或高度进行计算的，vw 和 vh 是根据视口 viewport 进行计算的，不会受父元素的影响。
    100vw 是包含了页面滚动条的宽度(页面滚动条属于视口的范围)
    在 PC 端使用 vw 单位时，如果超出了页面一屏的长度，出现了纵向滚动条，同时有元素设置了 width:100vw，则会导致出现横向滚动条，因为超出了 viewport 宽度

    移动端滚动条不占位，所以一般不会出现这个问题。所以尽量使用 100%。

    移动端拨打电话

    1. 在 head 标签中设置允许识别电话
      <meta name="format-detection" content="telephone=yes"/>
    2. 在 a 标签的 href 中添加电话号码
      <a href="tel:4008006821">联系客服</a>

    经测试，在微信中设置 telephone=no 也可以唤起电话

    微信中会出现屏蔽的方式，在电话后面添加一个锚点#mp.weixin.qq.com，如tel:10086#mp.weixin.qq.com

    在 VantUI 中使用 rem+vw 的方案

    根元素的 font size 使用 vw 去计算，不使用 JS，因为 VantUI 根据 375px，所以根元素的基础 font-size 是 37.5
